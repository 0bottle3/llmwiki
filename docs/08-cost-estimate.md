# 08. 비용 추정

전제: 10명 팀, 월 신규 1,000개 문서, 월 검색 10,000회, ap-northeast-2.

|------|------------------------|-----------------------------------|
| EKS 노드 (한계 비용) | ~$90 | ~$60 |
| 벡터 스토어 | $0 (Qdrant on EKS) | ~$170 (OpenSearch Serverless) |

EKS 클러스터 자체 비용($72/월)은 이미 운영 중이라고 가정하여 제외.

## 시나리오별 산정 근거

### EKS 노드 (한계 비용)

- search-api: 2 replica × 512Mi → 기존 노드 여유 활용
- processor: **CronJob(매시간)** → idle 시 파드 0, 스케줄 시에만 기동 → 2~3노드 × t3.medium 1~2시간/일
- curator: 무시 가능
- 한계 비용: 노드 0.5대 상시 + burst → **$60~90/월**

### 벡터 스토어

#### Qdrant on EKS (MVP 기본)

- 1~2 Pod, 512Mi~2Gi
- 기존 노드에 흡수 가능 → **$0 추가**
- 운영 부담 + 가용성은 사용자 책임

#### OpenSearch Serverless (확장 경로, Phase 2+)

- 최소 2 OCU (Indexing 1 + Search 1) = `0.24/OCU·시간 × 2 × 730시간 ≈ $350`
- 단, AOSS는 idle 시 자동 0.5 OCU로 떨어짐 (실제 청구 더 적음)
- 보수적으로 **$170/월**
- 데이터 양 늘면 +Storage ($0.024/GB·월)
- 운영 부담 없이 완전 관리형이 필요할 때 전환 검토

### S3

- 원본 + 가공본 + 임베딩 ≈ 50GB
- Standard: $0.025/GB·월 = $1.25
- 요청 (PUT/GET): ~$2
- Glacier IR 전환 후 더 낮아짐
- **$5/월**

### CloudWatch (SQS 제거)

- SQS: MVP에서 제거 (CronJob 방식으로 전환) → $0
- CloudWatch Logs: 5GB/월 ~$2.5
- 메트릭: 10개 커스텀 ~$3
- **~$3/월**

### Anthropic Claude (가공)

전제:
- 신규 문서 1,000개 × 평균 3,000 토큰 input + 1,000 출력
- Haiku 4.5 ($0.80/1M in, $4/1M out) + Batch API 50% + Prompt Caching 90%

```
입력 토큰: 1,000 × 3,000 = 3M
  - 캐시 적중 70%: 2.1M × $0.08 (cached) = $0.17
  - 캐시 미적중 30%: 0.9M × $0.80 = $0.72
출력 토큰: 1,000 × 1,000 = 1M × $4 = $4
배치 50% 할인: ($0.89 + $4) × 0.5 = $2.45
```

여기에 큐레이션/품질 점수/병합 판정 등 추가 호출 → 보수적으로 **$30~80/월**.

### Anthropic Claude (검색 리랭킹, 옵션)

- 월 10,000 검색 × 평균 5,000 토큰 (top-20 후보)
- Haiku: 50M × $0.80 = $40
- 캐싱 미적용 (쿼리마다 다름) → 비싸 보임
- 리랭킹은 **선택**, 안 써도 OK
- 쓸 거면 **$10~40/월**

### Voyage 임베딩

- 신규 문서 1,000개 × 1,000 토큰 = 1M tokens × $0.06 = $0.06
- 쿼리 임베딩 10,000 × 50 토큰 = 0.5M × $0.06 = $0.03
- 재인덱싱(분기 1회): 전체 ~ 5M tokens × $0.06 = $0.30
- **$5~10/월** (여유)

### ALB

- 시간당 $0.0225 × 730시간 = $16.4
- LCU 사용 ~$5
- **$25/월**

### NAT Gateway

- 외부 LLM API 호출용
- 시간당 $0.045 × 730 = $33
- 데이터: 토큰 트래픽 작아서 $1 미만
- VPC Endpoint Interface로 일부 대체 → **$30/월**

### Secrets Manager / KMS

- 시크릿 ~10개 × $0.40 = $4
- API 호출: 무시 가능
- KMS 키 1개 × $1 + 호출 ~$0.5
- **$5/월**

여전히 사람 1명 인건비보다 한참 싼 수준.

## 비용 절감 레버

1. **Prompt Caching 무조건 켜기** — 70% 이상 절감
2. **Batch API 우선** — 50% 추가 할인, 실시간성 거의 필요 없음
3. **Haiku 최우선** — 가공 품질 충분, Sonnet은 검증/리랭킹만
4. **CronJob이라 idle 시 파드 0** — 스케줄 시에만 기동, 별도 오토스케일러 불필요
5. **Qdrant on EKS (MVP 기본)** — OpenSearch Serverless 대비 월 $170 절감 (운영 부담 trade-off)
6. **NAT → VPC Endpoint** — LLM API용은 Anthropic이 PrivateLink 미지원이라 한계
7. **embedding 모델 차원 줄이기** — Voyage 1024 → 512 (저장/RAM 절반)

## 비용 모니터링

```
CloudWatch Alarms:
  - 일일 LLM 토큰 사용량 > 임계
  - 일일 S3 객체 수 > 임계
  - CronJob 실패 횟수 > 0

월말 자동 리포트:
  - curator가 토큰 사용량 × 단가로 모델별 비용 추정
  - 가장 비싼 사용자/주제 top 10
  - Prompt cache hit rate
```

## AI 대화 Transcript 수집 시 비용 주의

> **전제 재검토 필요**: 위 추정은 "월 신규 1,000개 문서" 기준이다.
> 팀원 전원의 AI 대화 transcript를 수집하면 메시지 수가 **수만 건/월로 폭증**하여 가공 LLM 비용이 몇 배가 될 수 있다.

### 민감도 추정

| 수집 범위 | 월 메시지 수 (예시) | 가공 LLM 비용 배율 |
|-----------|--------------------|--------------------|
| 일반 문서 1,000개 (기준) | ~1,000개 | ×1 (~$30~80) |
| 10명 × 일 평균 50 메시지 | ~15,000개/월 | ×3~5 (~$100~400) |
| 10명 × 적극 사용 (100+/일) | ~30,000개+/월 | ×5~10 (~$150~800) |

### 권장 대응

1. **Transcript는 opt-in/선별 수집** — 팀원이 명시적으로 공유 동의한 세션만 가공 대상
2. **증분 가공은 일반 API** (소량 문서), **대량 시드만 Batch API** (50% 할인) 활용
3. **가공 전 필터링** — 짧은 메시지, 반복 패턴, 코드 전용 세션은 가공 생략 옵션
4. **월별 토큰 cap** 설정 → CloudWatch 알람으로 임계 초과 시 가공 일시 중단

> 요약: transcript 전체 수집은 비용 ×3~10 가능. MVP 단계에서는 opt-in + 선별 수집을 강력히 권장.

## 결론

- **MVP**: Qdrant on EKS → 월 **~$200대**
- **확장 경로 (Phase 2+)**: OpenSearch Serverless → 월 ~$350~400
- 10명 팀이면 1인당 $20~40/월 = 노션 비즈니스($24)보다 약간 비싸지만
  **AI 통합 + 무한 LLM 사용**까지 합치면 압도적 가성비
- transcript 전체 수집 시 LLM 가공 비용 ×3~10 가능 → opt-in 수집 권장
